Apri Gateway

Principi tecnici e legali

FAIL-CLOSED: se una verifica (hash/firma/manifest/policy) fallisce o è indeterminata, l’output è invalidato e l’azione è bloccata.

HASH-ONLY: nessun documento nel gateway, solo impronte verificabili.

AUDIT-FIRST: ogni output deve essere verificabile offline in modo indipendente.

Ecosistema (mappa breve)

• IPR (Identity Primary Record): record tecnico minimo verificabile nel tempo
• UNEBDO: fondazione di validità tecnica giuridico-computabile
• OPC: policy computabile (valutazione e controllo)
• IPR stack: core / gate / comply / exchange
• Registry: pubblicazione sperimentale hash-only
• GitJoker-C2: publishing & compliance control su GitHub
